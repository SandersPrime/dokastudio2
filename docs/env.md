# Окружение и конфигурация

## Переменные окружения

Все переменные окружения должны быть определены в файле `.env`. Для настройки используйте `.env.example` как шаблон.

### Сервер

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `PORT` | Порт, на котором запускается сервер | `3000` |
| `NODE_ENV` | Окружение: `development`, `production` | `development` |
| `CLIENT_URL` | URL фронтенда для CORS | `http://localhost:3000` |

### База данных

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `DATABASE_URL` | Строка подключения к базе данных | `postgresql://user:password@localhost:5432/dokastudio` или `file:./dev.db` для SQLite |

### Безопасность

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `JWT_SECRET` | Секрет для подписи JWT-токенов | `your_jwt_secret_key_here` |
| `JWT_EXPIRES_IN` | Время жизни токена | `24h` |

### Загрузки

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `UPLOAD_DIR` | Директория для загрузок | `public/uploads` |
| `MAX_FILE_SIZE` | Макс. размер файла в байтах | `10485760` (10MB) |
| `ALLOWED_MIME_TYPES` | Разрешённые MIME-типы | `image/jpeg,image/png,audio/mpeg,video/mp4` |

### Сессии

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `SESSION_MAX_AGE` | Время жизни сессии в миллисекундах | `86400000` (24 часа) |

### Socket.IO

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `SOCKET_CORS_ORIGIN` | Разрешённый источник для сокетов | `http://localhost:3000` |

### Admin

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `ADMIN_EMAIL` | Email администратора | `admin@example.com` |
| `ADMIN_PASSWORD` | Пароль администратора | `admin123` |

## Режимы окружения

### Development
- Использует SQLite (`dev.db`) по умолчанию
- Автоматический запуск seed-скрипта
- Подробное логирование
- Включён nodemon

### Production
- Требует PostgreSQL
- Отключён seed-скрипт
- Минимизированные логи
- Оптимизированный запуск через `node app.js`

### Staging
(Опционально, при необходимости)
- Отдельная БД
- Ограниченный доступ
- Мониторинг

## Настройка

1. Скопируйте шаблон:
```bash
cp .env.example .env
```

2. Отредактируйте `.env`, указав актуальные значения:
```bash
nano .env
```

3. Убедитесь, что файл `.env` не попадает в git (проверьте `.gitignore`).

## Best Practices

- Никогда не коммитьте `.env` в репозиторий
- Используйте разные значения для разных окружений
- Храните `JWT_SECRET` в секрете и регулярно меняйте
- Ограничьте размер и типы загружаемых файлов
- Используйте HTTPS в production
- Регулярно обновляйте зависимости
